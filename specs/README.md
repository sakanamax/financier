# 專案規範與說明文件 (Specifications)

本目錄 (`specs/`) 用於統一存放專案的各式開發規範、準則與說明文件，以確保開發與維護時有一致的標準可循。

## 📄 包含文件

* **[chinese-comment-style.md](./chinese-comment-style.md)**：中文註解風格指南。規範程式碼中撰寫中文註解時的格式與語氣。
* **[dev-spec.md](./dev-spec.md)**：開發規範。涵蓋專案整體的開發流程、架構與程式碼撰寫標準。
* **[presentation-spec.md](./presentation-spec.md)**：簡報與展示規範。針對成果展示或專案相關簡報格式的準則。
* **[qa-spec.md](./qa-spec.md)**：測試與品管 (QA) 規範。定義測試流程、驗收標準及品質管控的要求。
* **[todo-spec.md](./todo-spec.md)**：待辦事項規範。說明如何撰寫與管理專案中的 TODO 及任務追蹤項目。

---
*保持規範文件的更新與落實，有助於提升專案的長期維護性！*
